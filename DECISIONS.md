# DECISIONS

Dated one-liners for every nontrivial call, per SPEC §0.

- 2026-06-12: Core sim works in raw Dota units (MS 300, ranges 600, etc.); renderer divides by 100 for world units. Dota numbers port verbatim, `tuning.ts` scales globally.
- 2026-06-12: Added two generic composition tools to the effect vocabulary instead of spending exotic slots: `repeat` (count/interval/sub-effects) and target selectors (e.g. `random-enemy-in-radius`, `random-point-in-ring`). Omnislash, Freezing Field, and Chain Frost all compose from primitives; zero exotics spent in Phase 1.
- 2026-06-12: Trigger system generalized: `on-cast`, `on-damage-taken`, `on-attack-land`, `on-kill`, `on-nearby-death`, `on-nearby-enemy-cast` — the spec's listed mechanic flags (Blink lockout, Aftershock, Flesh Heap, Magic Wand) are instances of one generic trigger primitive.
- 2026-06-12: Ability skill points auto-assign on level-up (ult at 6/12/18, basics round-robin via per-hero `skillOrder`). Manual skilling adds UI without Phase-1 value; talents (10/15/20/25) stay manual per spec.
- 2026-06-12: Phase 1 recruitment: non-starter heroes stand at lore spots in Tranquil Vale and join via a Binding Sigil interaction (right-click). The full Find→Trial→Bind chain is Phase 2 scope; this placeholder is replaced then (recruitment framework is not on the P1 checklist; mid-fight hero swap requires a 2nd hero, so some recruitment path must exist in P1).
- 2026-06-12: Power Treads cut from P1 item list in favor of Arcane Boots (tread-switching micro deserves real treatment later; Arcane Boots' mana-battery identity survives intact).
- 2026-06-12: Party wipe in overworld: respawn at town shrine, lose 10% gold (Diablo-style death tax), wild camps the player left reset via normal respawn timers.
- 2026-06-12: Wild creeps leash-reset (return to camp and heal to full) beyond ~1800 units from camp, preventing drag-cheese.
- 2026-06-12: Cleave deals full pre-armor physical to secondary targets (canon-faithful); crits use plain seeded RNG, not pseudo-random distribution (simplest implementation that works).
- 2026-06-12: Attack projectiles always land on arrival (not disjointable) in P1; spell projectiles disjoint on blink/invis/cyclone per canon.
- 2026-06-12: Capture thresholds by tier: small 30%/2.5s, medium 25%/3.0s, large 20%/3.5s, ancient 15%/4.5s — all in tuning.ts.
- 2026-06-12: Entourage creep death: creep returns to storage "fainted" for 90s (tuning), then fieldable again. Keeps death meaningful without dead content.
- 2026-06-12: Phase 1 starts with a 2600g Dawnshade stipend (`TUNING.startingGold`) so the acceptance demo can buy Blink immediately; long-term economy balance is Phase 2+.
- 2026-06-12: Added a tutorial kobold camp and moved Pudge near Dawnshade so capture, companion fielding, recruitment, and 1-5 swap are reachable in the first minute.
- 2026-06-12: Save imports and slot loads now validate version, region/hero/creep references, party bounds, and core shape before starting a game.
- 2026-06-12: Map mode uses procedural in-world markers for town, shrine, camps, and recruitable heroes as the Phase 1 far-readability layer instead of a separate minimap UI.
- 2026-06-12: Added Luna, Sven, and Axe as data-only heroes with no exotic slots; placed them in Tranquil Vale temporarily so the one-region build can recruit/test them before their lore regions exist.
- 2026-06-12: Added spell amplification and status resistance to the stat vocabulary for Kaya/Sange identity instead of treating those items as cosmetic stat sticks.
- 2026-06-12: Increased procedural model tessellation and switched unit materials to smooth Lambert shading; preserves asset-free stylization while reducing placeholder-low-poly jaggedness.
- 2026-06-12: Combat feel pass keeps one shared core: faster AI cadence, role-based macro formations, taunt as forced attacks, and a damage-threat boss controller for 5v1 raids.
- 2026-06-12: Phase 2 echo economy starts as per-hero persisted `EchoProgress`: first owned-echo kill unlocks facet swapping, then one talent tier at a time activates the opposite branch through the generic `buildHero` path.
- 2026-06-12: Phase 2 region travel hot-swaps the single loaded region through the existing save/load event path; this preserves the current renderer architecture while enabling a three-region continent.
- 2026-06-12: Recruitment trials are data-defined per hero and use a generic Find→Trial→Bind interpreter; the bind step is a real enemy hero echo fight, while the six bespoke trial kinds stay lightweight and reusable for Phase 2.
- 2026-06-12: Gyms reuse the headless macro sim as best-of-3 sessions; Captain Calls are simulated as timed player-control windows that issue clutch orders, keeping gym logic testable without a second renderer.
- 2026-06-12: Spec adds a bonus Phase 5 (Resonance, Feel & Fidelity). Resonance is an opt-in Genshin-style elemental layer (7 elements mapped from canon, a generic reaction-table resolver, party-composition resonance buffs) built entirely from the existing status/trigger/aura vocabulary — zero exotic slots, micro+raids only, gyms/Elite Five stay pure Dota.
- 2026-06-12: Crunch mode — all hard restrictions removed from SPEC. §0 phase-gating/dependency/perf/test mandates, §1 "load-bearing rules" framing, §2 exotic (~25) and VFX (~12) caps, §5 "cut don't water down" / "never purchasable" / min counts, §7 ≤8-gambit cap, the no-physics/navmesh and no-copy lines, and §10 Non-Negotiables are now non-binding aspirations. Headless core, data-driven content, and Dota feel fidelity stay as recommended defaults, not requirements.
- 2026-06-12: Phase 5 reverses the procedural-only / no-external-assets rule (finalized decision): heroes get detailed rigged models that resemble (not replicate — exact copies are neither possible nor wanted) their Dota 2 counterparts via a real asset pipeline (GLTFLoader, textures, skeletal animation, post-FX), plus a feel pass (attack-move/stop with shift-queue, per-hero attack animation timed to attackPoint/BAT, real positional SFX, floating combat text). Core stays headless (§1.1); written content stays original (§10). Procedural primitive models remain the P1–P4 placeholder path.
- 2026-06-12: Phase 3 keeps the existing 20 Phase 2 heroes, including Sniper/Jakiro/Omniknight/Windranger, and adds the 44 missing Phase 3 heroes as data-only entries. This yields 64 registered heroes, preserving working fixtures while satisfying the ≥60 acceptance floor.
- 2026-06-12: Planned Phase 3 exotics are registered as no-op data hooks first (`invoke`, `chronosphere`, `stone-gaze`, `reincarnation`, `rearm`, four raid signatures, and `refresh-cooldowns`). The primitive behavior remains testable now; bespoke presentation/implementation can deepen later without changing data IDs.
- 2026-06-12: Boss tier scaling starts at Normal 1.0x, Nightmare 1.65x HP / 1.28x damage, and Hell 2.45x HP / 1.65x damage. Nightmare opens after the regional badge; Hell opens after a cleared Nightmare rerun.
- 2026-06-12: Gated top-tier items are registered as normal `ItemDef`s for inventory/combat resolution but are excluded from all normal shops; Secret Shops sell only components, while bosses and raids own assembled drop chances plus pity.
- 2026-06-12: Neutral items are their own registry with a dedicated save slot/stash. Gold can reroll, reclaim, or enchant three duplicates up-tier, but no gold sink directly sells a neutral or gated top-tier item.
- 2026-06-12: Save v3 adds Phase 3 progression fields and migrates v2-shaped saves by defaulting new maps/stashes/counters and hero neutral slots.
- 2026-06-12: Elite Five draft data uses placeholder original homage names and deterministic pick/ban construction over the recruited roster; Champion remains a marquee 5v5 data team.
- 2026-06-12: Optimization pass starts with a headless perf budget test and at-scale determinism tests before deeper refactors. The stress case covers 30 units / 200 projectiles plus mixed zones and resonance so the §0 budget has an automated guard.
- 2026-06-12: Core sim broadphase is a deterministic uniform grid rebuilt at fixed tick points. Unit-array removal keeps stable splice semantics for now to avoid mid-tick iteration behavior changes; local queries, movement collision, projectiles, zones, auras, cleave, and nearest-enemy helpers route through the grid.
- 2026-06-12: Fixed-step render-frame catchup is capped by `TUNING.maxSimTicksPerFrame`; a bad frame degrades to slow-mo instead of spiraling through unbounded sim ticks.
- 2026-06-12: Unit stat refresh is dirty/deadline cached: stat sources (level, ability levels, items, external/permanent mods, status list) gate expensive derivation, while invis fade deadlines still force time-sensitive summary refreshes. Direct setup/test mutations are caught by a cheap source signature in `Unit.refresh`.
- 2026-06-12: Procedural model geometry is canonicalized through the model builder's `mesh()` helper. Materials remain per-rig for invis/death opacity safety, while repeated primitive geometry is shared across rigs.
- 2026-06-12: VFX now canonicalize reusable geometry and compact transient effects in place; dynamic beam/line geometries stay per-effect because their lengths/points vary. Projectile sync reuses a Set between frames.
- 2026-06-12: HP bars use shared plane-geometry mesh fills instead of one canvas/texture per unit. Renderer quality tiers now own DPR caps, shadow map size/type, shadow enablement, and transient VFX caps.
- 2026-06-12: Phase 5 Resonance ships as an opt-in save setting (`settings.resonance`) defaulted off. Hero/ability elements are stamped during content registration from canonical identity maps, keeping source data compact while letting data lint prove coverage. Macro gyms remain unaffected because only overworld `Game` enables `sim.resonanceEnabled`.
- 2026-06-12: Reactions resolve in the central damage path from one generic table. Damage multipliers, extra magical bursts, Freeze, Superconduct armor shred, Swirl/Crystallize/Burning hooks, and item on-hit elements all reuse statuses/statmods/events instead of adding hero-specific exotics.
- 2026-06-12: Phase 5 feel pass exposes existing core `attack-move` via `A`-click and adds a small Game-level shift order queue. This keeps the sim order model simple while matching Dota/RTS muscle memory for move/attack/cast/item chains.
- 2026-06-12: Phase 4 SFX closure uses a tiny event-driven procedural WebAudio layer first; Phase 5 real sound assets can replace the generator later without changing sim events.
- 2026-06-12: Phase 5 graphics pipeline starts with a starter-hero GLB manifest and `GLTFLoader` cache plus procedural fallback. Missing assets never break the playable build; dropping GLBs into `/assets/heroes/` is the next fidelity step.
- 2026-06-12: Phase 5 visual fidelity is not only a loader stub: the renderer now applies hero-specific procedural likeness overlays for the shipped starter roster (mask/crest, fur hood/crown, butcher belly/scar/apron, totem bruiser shoulders/beard, dwarf rifleman helm/goggles/beard, skeletal frost king crown/cape) and locks them with lint coverage.
- 2026-06-12: Item fidelity now includes additional wearable geometry beyond weapon swaps: Crystalys shards, Scythe hex sigil, Aghanim mana orb, plus existing boots trails, armor pauldrons, Skadi shards, Heart core, Butterfly wing blades, Rapier glow, and Maelstrom storm haft. Attack animation branches are hero/range aware while still timed from `attackPoint`/BAT.
- 2026-06-12: Procedural SFX upgraded from placeholder beeps to archetype/reaction-aware WebAudio envelopes (casts, hits, crits, heals, items, death, capture/level chimes, Vaporize/Melt/Freeze/etc.). Browser QA must stay console-clean; replaced deprecated `PCFSoftShadowMap` with `PCFShadowMap` and added an inline favicon.
- 2026-06-12: Presentation-spec reward loop starts with one Game-layer `gold` event for kills, last-hits, XP overflow, echo surplus, and sells. HUD/audio share a 1.5s reward-streak window; coin audio climbs by semitone steps, and last-hits get the brighter `lasthit` treatment without touching core sim state.
- 2026-06-13: Phase 6 kickoff. Phase 6 wires the systems Phases 2–4 shipped as helpers/data-only. Build order follows the spec §10 milestones, each committed green.
- 2026-06-13: Save v4 (Phase 6 §3.15). `SAVE_VERSION` 3 -> 4. New `GameSave` fields: `reputation` (karma, default 0), `codexUnlocks: string[]`, `journalSeen: string[]`. `settings` now holds an `audio` channel object `{ master, sfx, voice, stinger, muted }` plus `quickcast`/`resonance`/`minimap`. v3->v4 migration folds the loose volume fields: `masterVolume->master`, `sfxVolume->sfx`, `musicVolume->stinger` (stingers are the musical layer); `voice` has no v3 analogue so it defaults to 0.7. Migration chains v2 -> migratePhase3Save -> migratePhase4Save -> v4 and preserves an existing `audio` object so v4 saves reload identically. `migrateSave`/`validateSave` accept v2/v3/v4.
- 2026-06-13: Recruit level ceiling curve = `[15, 22, 30]` indexed by badge count and clamped at the last entry (matches the §3.4 formula `[15,22,30][min(badge,2)]` and the §6.8 named acceptance values; ≥2 badges stays at the level-30 cap, satisfying Phase 3 §4.6 "rises to the cap"). Stored in `TUNING.recruitLevelCap`, exposed as the pure `recruitLevelCap(badgeCount)` in `core/progression.ts`. XP past the ceiling banks in `Unit.addXp(amount, levelCap)` (xp accumulates up to level-30 xp, level clamps to the ceiling); a new badge calls `applyRecruitCeiling()` so banked XP catches up. Trial relocation shard floor = 1 (`TUNING.relocationShardFloor`); default Find shards = 2.
- 2026-06-13: Reputation scale is a small signed integer clamped to [-20, 20], starting at 0. Good-karma recruit gate opens at `reputation >= 3` (`TUNING.reputationGoodGate`, applied to Omniknight's templated trial). The Souls Pact (Shadow Fiend) greed path subtracts 4 (`TUNING.reputationSoulsPactDrop`); the honor path adds 2 (`TUNING.reputationHonorGain`).
- 2026-06-13: Echo fidelity (§3.2/§3.3). `core/echo-unit.ts#spawnHeroEchoUnit` is the single echo builder for overworld echoes and trial echoes: full kit via `buildHero`, enemy team, **gambit controller** (`buildDefaultGambit`), ×0.6 max HP (`TUNING.echoHpTaxPct = 0.4`), no item slots, `Unit.isEcho` flag (renderer draws translucent at 0.5 opacity). Overworld echoes set `ctrl.leashRadius` (`TUNING.echoLeashRadius = 2200`) — a new optional gambit leash so they don't roam the whole region; macro/raid gambit units leave it unset and are unaffected.
- 2026-06-13: TrialRunner (§3.1). `core/trials.ts` maps each `TrialKind` to one of nine mechanics (duel/endure/cull/hit/combo/assassinate/convert/fetch/choice/gated) sharing spawn+observe+tick code. It spawns into the live `Sim`, consumes `SimEvent`s via `observe`, and resolves `complete`/`fail` on `tick`. Added two repo kinds: `souls-pact` (Shadow Fiend greed/honor choice) and `stealth-hunt` (Riki endure). `trialGateOpen(trial, ctx)` enforces `reputationGate`, roster-legend (≥50 recruited), and raid-recruit (≥1 raid clear). The runner drives both the live `Game.tryRecruit` flow and headless acceptance tests (which feed scripted events + time).
- 2026-06-13: Trial relocation policy: on `fail`, shards reset to the floor (1) and the marker `stage` returns to `unfound` (a rumor again, never locked); the NPC marker moves to the next entry of `TrialDef.relocateSpots` (two alternate in-region spots per trial, cycled). Marker position is in-session only (a reload respawns the NPC at its data spot with shards at the floor).
- 2026-06-13: Added a headless `Game` seam (`Game.headless(save)` + `SceneLike`/`AudioLike` interfaces + `HeadlessScene`/`HeadlessAudio`) so the orchestrator runs in node tests without WebGL/WebAudio. The real `new Game(canvas, save)` path is unchanged. This unblocks Game-level integration tests for Phase 6 (echo-advances-find, relocation, recruit gate, and the M3–M10 systems).
- 2026-06-13: M3 gym fights become **live + stepped** via `LiveGymFight` (`systems/macro-session.ts`): it owns the per-round macro `Sim`, a player `CaptainCallController(team 0, 3 charges)` and an enemy `CaptainCallController(team 1, 3 + gym.enemyBonusCaptainCalls)`, and an auto-call AI (spaced every 12s from 4s, prefers an ult-ready hero). `runGymMatch` now runs a `LiveGymFight` headlessly with `autoPlayer:true` so both the auto-resolve path and the new live path share one engine and the enemy always gets its bonus calls. Live path: `Game.startLiveGym` swaps the rendered scene to the gym sim (with `scene.resetUnitViews()` on enter/exit so overworld uids never alias gym uids), the overworld sim freezes underneath, and `Game.update` takes a dedicated `updateLiveGym` branch. The player spends a call via a HUD button or Space (`Game.liveGymPlayerCall`), taking an ult-ready hero to player control for `captainCallSec`; the call auto-steers to the lowest-HP enemy then reverts and decrements. Both teams' charges + the round score render in a live overlay. Best-of-3 is preserved per round.
- 2026-06-13: The gambit editor (Phase 2 §3.5) is a per-hero modal rule builder over the existing `GambitRule` grammar: an ordered, reorderable list of ≤8 rules, each built from a condition `<select>` (+ numeric param) and an action `<select>` (+ slot 0–3 and target-mode for casts, item id for use-item). Rules support multiple AND conditions (chips with add/remove) to preserve what the presets author. Up/down reorder + delete per row; first match wins (the controller's existing behavior). The Default/Aggro/Safe presets populate the builder instead of replacing it. The gym **pre-fight screen** reuses this: pick is the existing party of 5, set each hero's gambits, then Fight Live or Auto-Resolve; M5 reuses the same pre-fight for the Elite draft.
- 2026-06-13: M4 boss reruns (§3.6) run as a live 5v1 on the raid sim (`Game.runBossFight` → `runRaidBattle`) with the party's authored gambits/items; the boss hero is leveled (28 boss / 24 mini), armed (BKB + Assault Cuirass), and scaled by `tierScale(tier)` over the base `raidBossHpScale`/`raidBossDamageScale`. Difficulty progress stores `{ tier: <highest cleared>, dryClears: <pity streak> }`: a Normal clear opens Nightmare (badge + a progress entry), and Hell opens only after a Nightmare clear that reset the streak — exactly what the tested `bossTierUnlocked` encodes (`tier==='nightmare' && dryClears===0`). The region badge gate uses the **boss's** region gym, not the current region. Loot uses `rollLoot` with the running streak + `bossLootSeed`; guaranteed components and any assembled drop go to `inventoryStash` (+ `heldUniques`).
- 2026-06-13: M4 neutral items (§3.7). Wild-creep deaths roll `rollNeutralDrop(tier, ALL_NEUTRAL_ITEMS, seed)` in `handleKillCredit` into `neutralStash` (count-stacked). A hero's dedicated `neutralSlot` is separate from the six sellable item slots and **cannot be sold** — `equipNeutral`/`reclaimNeutral` only move it between slot and stash, never to gold. Neutral `passiveMods` apply to the live unit through a tracked `RosterEntry.neutralMods` add/subtract pass (mirrors resonance/day-night) so swaps are clean; auras/actives are out of scope (no shipped neutral defines one — the Dota-style `n` key is reserved and messages "passive"). Tinker's Bench: `tinkerReroll` (in-tier, `rerollCost`), `tinkerEnchant` (3 dupes → `enchantsInto`, one tier up, `enchantCost`), both town-gated.
- 2026-06-13: M4 gold sinks (§3.8) are callable `Game` actions incrementing `goldSinks`: `buyback` (instant revive of a fallen hero, `buybackCost(level, buybacks)`; the active hero respawns in place, a benched hero becomes swap-ready — the existing party-wipe shrine-revive is unchanged), `buyTome` (`tomePurchase`, diminishing XP, banks under the recruit ceiling), `respec` (out-of-combat, `respecCost`, clears talent picks and re-spawns the unit to drop talent mods), and `healParty` (town, `healServiceCost`). No sink vends items; the gated top-tier set (`GATED_TOP_TIER` in `systems/game.ts`) is rejected by `buyItem`/`shopSells`, which now also require an item to belong to the town shop or a nearby secret shop. Live kill rewards already route through `scaledBounty` (region/tier/creep-tier/star) with post-cap XP→gold via `overflowXpToGold`; M4 adds the integration test the earlier phase skipped.
- 2026-06-13: M4 surfaces everything through a single **Town Services** modal (`Y` key, `modalKind:'services'`) with three sections — Boss Reruns (per-region bosses with Normal/Nightmare/Hell buttons gated by `bossUnlockedTiers`), Tinker's Bench (stash reroll/enchant + per-hero slot equip/reclaim), and Recovery & Growth (Heal, Buyback, per-hero Tome/Respec). One consolidated panel keeps the wiring reachable without new world-marker plumbing; boss markers in `RegionDef` can replace the boss section later without changing the `Game` API.
- 2026-06-13: M5 raid mechanics (§3.9) execute via `core/macro.ts#runRaidEncounter`: it builds the 5v1 with `raidSetupFromDef` (tier-scaled boss) and steps the sim, firing one unique instance per scripted beat. Beats keyed off boss HP%: `addWaves[].summon × count` spawn on the boss team near the boss; `zones` drop a telegraphed damage/aura zone on the living-party centroid (dodgeable); a signature beat at 50% composes a wide damage zone (the registered signature exotic stays a presentation no-op). The enrage is a **timer** (`enrageSec`), not an HP threshold — on expiry the boss gets +120% damage / +120% attack speed / +30% move speed. Threat + taunt are already handled by the `boss` controller used in the raid sim, so no extra wiring. `runRaidEncounter` returns a `fired[]` mechanic log + `cleared`; an opt-in `captureEvents` keeps the sim event history for tests.
- 2026-06-13: M5 death handling lives in the shared `runBattleToResult` so every macro/raid fight gets it. **Rapier-on-death** (§4.2): when a hero holding `divine-rapier` dies, the item is removed from the corpse and handed to the nearest living enemy hero (claimed for the round); recorded in `MacroResult.rapierDrops`. **Aegis** (§3.9): an opt-in `aegisTeam` flag makes the first hero on that team to fall auto-revive once via the new `Sim.reviveUnit(u, hpPct, manaPct)` primitive (flips a lingering corpse back to alive in place, clears statuses/orders, restores hp/mana, marks the spatial grid dirty, emits a `revive` event); recorded as `aegisConsumed`. `reviveUnit` is reusable for Reincarnation later.
- 2026-06-13: M5 Roshan specifics (§3.9) live in `Game.runRaid`/`deliverRaidLoot`. The loot table's guaranteed Aegis becomes the held one-use charge (`raidProgress[id].aegisHeld`) rather than a stash item; `aegisReady()` carries it into the next raid and `runRaid` consumes it when the encounter reports `aegisConsumed`. Killing Roshan sets `roshanRespawnAt = playtime + TUNING.roshanRespawnSec` (480s) and blocks re-pulls until it elapses; clear #`TUNING.roshanRepeatDropFromClear` (2) onward also drops a Refresher Shard + Cheese. Other raids deliver loot through the normal path with pity. Raids are reachable from the Town Services "Raids" section (full party; Roshan respects its timer); they auto-resolve with mechanics firing rather than rendering a live raid — the live-stepped path (à la `LiveGymFight`) is a presentation follow-up, the headless encounter is the system of record.
- 2026-06-13: M5 Elite Five gauntlet (§3.10): `Game.runEliteMatch` drafts the enemy per-member by passing a single-member `DraftDef` to `draftTeams` (reuse), runs a `runMacroBattle` 5v5, advances `eliteFive.defeated` on a win, and **leaves it unchanged on a loss** so the player re-challenges that same member (never a hard lockout, never a reset to 0). `runChampion` is gated behind `defeated >= 5` and flips `eliteFive.championDown` on a win (`ELITE_DRAFT.champion` is a marquee 5v5). Both accept an optional scripted `playerTeam` (used by the winnability test); from the UI they draft from the recruited roster / field the current party. Surfaced as the Town Services "Conquest" section. Faction exclusivity (§3.10): the faction-choice trial records the pick through `chooseFaction(current, regionId, heroId, pair)` in `finishTrial`, and `tryRecruit` refuses a hero whose region already sided with the rival captain (`factionLockedHero`).
