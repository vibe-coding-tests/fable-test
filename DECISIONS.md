# DECISIONS

Dated one-liners for every nontrivial call, per SPEC §0.

- 2026-06-12: Core sim works in raw Dota units (MS 300, ranges 600, etc.); renderer divides by 100 for world units. Dota numbers port verbatim, `tuning.ts` scales globally.
- 2026-06-12: Added two generic composition tools to the effect vocabulary instead of spending exotic slots: `repeat` (count/interval/sub-effects) and target selectors (e.g. `random-enemy-in-radius`, `random-point-in-ring`). Omnislash, Freezing Field, and Chain Frost all compose from primitives; zero exotics spent in Phase 1.
- 2026-06-12: Trigger system generalized: `on-cast`, `on-damage-taken`, `on-attack-land`, `on-kill`, `on-nearby-death`, `on-nearby-enemy-cast` — the spec's listed mechanic flags (Blink lockout, Aftershock, Flesh Heap, Magic Wand) are instances of one generic trigger primitive.
- 2026-06-12: Ability skill points auto-assign on level-up (ult at 6/12/18, basics round-robin via per-hero `skillOrder`). Manual skilling adds UI without Phase-1 value; talents (10/15/20/25) stay manual per spec.
- 2026-06-12: Phase 1 recruitment: non-starter heroes stand at lore spots in Tranquil Vale and join via a Binding Sigil interaction (right-click). The full Find→Trial→Bind chain is Phase 2 scope; this placeholder is replaced then (recruitment framework is not on the P1 checklist; mid-fight hero swap requires a 2nd hero, so some recruitment path must exist in P1).
- 2026-06-12: Power Treads cut from P1 item list in favor of Arcane Boots (tread-switching micro deserves real treatment later; Arcane Boots' mana-battery identity survives intact).
- 2026-06-12: Party wipe in overworld: respawn at town shrine, lose 10% gold (Diablo-style death tax), wild camps the player left reset via normal respawn timers.
- 2026-06-12: Wild creeps leash-reset (return to camp and heal to full) beyond ~1800 units from camp, preventing drag-cheese.
- 2026-06-12: Cleave deals full pre-armor physical to secondary targets (canon-faithful); crits use plain seeded RNG, not pseudo-random distribution (simplest implementation that works).
- 2026-06-12: Attack projectiles always land on arrival (not disjointable) in P1; spell projectiles disjoint on blink/invis/cyclone per canon.
- 2026-06-12: Capture thresholds by tier: small 30%/2.5s, medium 25%/3.0s, large 20%/3.5s, ancient 15%/4.5s — all in tuning.ts.
- 2026-06-12: Entourage creep death: creep returns to storage "fainted" for 90s (tuning), then fieldable again. Keeps death meaningful without dead content.
- 2026-06-12: Phase 1 starts with a 2600g Dawnshade stipend (`TUNING.startingGold`) so the acceptance demo can buy Blink immediately; long-term economy balance is Phase 2+.
- 2026-06-12: Added a tutorial kobold camp and moved Pudge near Dawnshade so capture, companion fielding, recruitment, and 1-5 swap are reachable in the first minute.

Resonance is an opt-in Genshin-style elemental layer (7 elements mapped from canon, a generic reaction-table resolver, party-composition resonance buffs) built entirely from the existing status/trigger/aura vocabulary — zero exotic slots, micro+raids only, gyms/Elite Five stay pure Dota.

Core stays headless (§1.1); written content stays original (§10). Procedural primitive models remain the P1–P4 placeholder path.

- 2026-06-13: Recruit level ceiling curve = `[15, 22, 30]` indexed by badge count and clamped at the last entry (matches the §3.4 formula `[15,22,30][min(badge,2)]` and the §6.8 named acceptance values; ≥2 badges stays at the level-30 cap, satisfying Phase 3 §4.6 "rises to the cap"). - 2026-06-13: Echo fidelity (§3.2/§3.3). `core/echo-unit.ts#spawnHeroEchoUnit` is the single echo builder for overworld echoes and trial echoes: full kit via `buildHero`, enemy team, **gambit controller** (`buildDefaultGambit`), ×0.6 max HP (`TUNING.echoHpTaxPct = 0.4`), no item slots, `Unit.isEcho` flag (renderer draws translucent at 0.5 opacity). - 2026-06-13: TrialRunner (§3.1). `core/trials.ts` maps each `TrialKind` to one of nine mechanics (duel/endure/cull/hit/combo/assassinate/convert/fetch/choice/gated) sharing spawn+observe+tick code. It spawns into the live `Sim`, consumes `SimEvent`s via `observe`, and resolves `complete`/`fail` on `tick`. Added two repo kinds: `souls-pact` (Shadow Fiend greed/honor choice) and `stealth-hunt` (Riki endure). `trialGateOpen(trial, ctx)` enforces `reputationGate`, roster-legend (≥50 recruited), and raid-recruit (≥1 raid clear). The runner drives both the live `Game.tryRecruit` flow and headless acceptance tests (which feed scripted events + time).
- 2026-06-13: Trial relocation policy: on `fail`, shards reset to the floor (1) and the marker `stage` returns to `unfound` (a rumor again, never locked); the NPC marker moves to the next entry of `TrialDef.relocateSpots` (two alternate in-region spots per trial, cycled). - 2026-06-13: M3 gym fights become **live + stepped** via `LiveGymFight` (`systems/macro-session.ts`): it owns the per-round macro `Sim`, a player `CaptainCallController(team 0, 3 charges)` and an enemy `CaptainCallController(team 1, 3 + gym.enemyBonusCaptainCalls)`, and an auto-call AI (spaced every 12s from 4s, prefers an ult-ready hero). `runGymMatch` now runs a `LiveGymFight` headlessly with `autoPlayer:true` so both the auto-resolve path and the new live path share one engine and the enemy always gets its bonus calls. Live path: `Game.startLiveGym` swaps the rendered scene to the gym sim (with `scene.resetUnitViews()` on enter/exit so overworld uids never alias gym uids), the overworld sim freezes underneath, and `Game.update` takes a dedicated `updateLiveGym` branch. The player spends a call via a HUD button or Space (`Game.liveGymPlayerCall`), taking an ult-ready hero to player control for `captainCallSec`; the call auto-steers to the lowest-HP enemy then reverts and decrements. - 2026-06-13: The gambit editor (Phase 2 §3.5) is a per-hero modal rule builder over the existing `GambitRule` grammar: an ordered, reorderable list of ≤8 rules, each built from a condition `<select>` (+ numeric param) and an action `<select>` (+ slot 0–3 and target-mode for casts, item id for use-item). Rules support multiple AND conditions (chips with add/remove) to preserve what the presets author. Up/down reorder + delete per row; first match wins (the controller's existing behavior). The Default/Aggro/Safe presets populate the builder instead of replacing it. The gym **pre-fight screen** reuses this: pick is the existing party of 5, set each hero's gambits, then Fight Live or Auto-Resolve; M5 reuses the same pre-fight for the Elite draft.

Tinker's Bench: `tinkerReroll` (in-tier, `rerollCost`), `tinkerEnchant` (3 dupes → `enchantsInto`, one tier up, `enchantCost`), both town-gated.
- 2026-06-13: M4 gold sinks (§3.8) are callable `Game` actions incrementing `goldSinks`: `buyback` (instant revive of a fallen hero, `buybackCost(level, buybacks)`; the active hero respawns in place, a benched hero becomes swap-ready — the existing party-wipe shrine-revive is unchanged), `buyTome` (`tomePurchase`, diminishing XP, banks under the recruit ceiling), `respec` (out-of-combat, `respecCost`, clears talent picks and re-spawns the unit to drop talent mods), and `healParty` (town, `healServiceCost`). Live kill rewards already route through `scaledBounty` (region/tier/creep-tier/star) with post-cap XP→gold via `overflowXpToGold`; M4 adds the integration test the earlier phase skipped.

- 2026-06-13: M5 raid mechanics (§3.9) execute via `core/macro.ts#runRaidEncounter`: it builds the 5v1 with `raidSetupFromDef` (tier-scaled boss) and steps the sim, firing one unique instance per scripted beat. Beats keyed off boss HP%: `addWaves[].summon × count` spawn on the boss team near the boss; `zones` drop a telegraphed damage/aura zone on the living-party centroid (dodgeable); a signature beat at 50% composes a wide damage zone (the registered signature exotic stays a presentation no-op). Threat + taunt are already handled by the `boss` controller used in the raid sim, so no extra wiring. `runRaidEncounter` returns a `fired[]` mechanic log + `cleared`; an opt-in `captureEvents` keeps the sim event history for tests.

- 2026-06-13: M5 Roshan specifics (§3.9) live in `Game.runRaid`/`deliverRaidLoot`. The loot table's guaranteed Aegis becomes the held one-use charge (`raidProgress[id].aegisHeld`) rather than a stash item; `aegisReady()` carries it into the next raid and `runRaid` consumes it when the encounter reports `aegisConsumed`. - 2026-06-13: M5 Elite Five gauntlet (§3.10): `Game.runEliteMatch` drafts the enemy per-member by passing a single-member `DraftDef` to `draftTeams` (reuse), runs a `runMacroBattle` 5v5, advances `eliteFive.defeated` on a win, and **leaves it unchanged on a loss** so the player re-challenges that same member (never a hard lockout, never a reset to 0). `runChampion` is gated behind `defeated >= 5` and flips `eliteFive.championDown` on a win (`ELITE_DRAFT.champion` is a marquee 5v5). Surfaced as the Town Services "Conquest" section. - 2026-06-13: M6 animation system (§3.11). We standardize on the **in-repo** `AnimGesture` vocabulary already in `types.ts` (`melee-swing`, `ranged-shot`, `staff-cast`, `ground-slam`, `dash`, `channel-loop`, `summon-gesture`, `item-use`, `global-cast`), not the illustrative Phase 4 names. `core/gestures.ts` adds two total (never-throwing) resolvers: `gestureForAbility(def)` returns `def.anim` or infers from channel/summon/self-blink/global/zone/projectile/targeting (effects over vfx — `storm` vfx is deliberately *not* auto-ranged so melee whirls like Blade Fury don't mis-tag); `soundForAbility(def)` returns `def.sound` or infers from heal-effect + vfx archetype (reused by M7). On cast, `actions.ts` stamps `Unit.castGesture` (ability casts in `handleCastOrder`; item actives in `handleItemOrder`, which also opens a 0.35s `castingUntil` window so a pressed item reads on the body). The animator (`engine/animator.ts`) becomes a closed gesture player: `castPose(gesture, castStyle, time, weight)` poses arms/body per gesture, shaped by `animProfile.castStyle` and weighted by `rig.scale` (dainty vs heavy read differently); the old iconic `heroId` cast branches survive only as the fallback when `castGesture` is null (summons / sparse profiles). Generic attack windup is now weight-scaled too. `Unit.animProfile` is copied from the def at spawn (`Sim.spawnHero`, `makeCreepUnit`).
- 2026-06-13: M6 anim/sound coverage (§4/§6). The nine hand-authored heroes (`data/heroes/*.ts`) now carry intentional per-ability `anim`/`sound` + an `animProfile` (e.g. Pudge/Axe/Earthshaker `brute`, CM/Lich `caster`, Sniper `ranged`). Generated rosters default programmatically so the diff stays small: the `phase2.ts` `hero()` factory maps each ability through `gestureForAbility`/`soundForAbility` and derives `animProfile` from attribute/range; `phase3.ts` already tagged everything. Hand-authored Phase 1 creeps + item actives are normalized at their collection points (`ALL_CREEPS`/`ALL_ITEMS` `.map(...)`) — item actives default to `item-use` (self-displacers like Blink → `dash`) with an `item` sound; creeps default `animProfile` + ability tags. Data-lint (test 19) is tightened from "if present" to **required** on every hero/creep ability and item active, `animProfile` required on every hero/creep, and a coverage block proves the gesture player resolves a valid gesture+sound for every ability (including nested summon sub-abilities) without throwing. - 2026-06-13: Loading screen vs. the boot/travel lag spike (`§9.4`). The spinner animates via compositor-friendly `transform`/`opacity` (keeps moving during a main-thread block) and honors `prefers-reduced-motion`. A `minMs` floor prevents a flash on fast machines.
